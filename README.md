# Final Project: Персональная кулинарная книга

Выполнил студент группы Б9123-09.03.02 ПРС Агарков Егор Дмитриевич

Приложение для поиска, сохранения и организации рецептов.  
Разработано как финальный проект по курсу Android-разработки.  
**Offline-first**, с собственными пользовательскими данными, фоновыми задачами и тестами.

## Основные возможности

- Поиск рецептов через Spoonacular API с кэшированием
- Создание, редактирование и хранение собственных рецептов
- Коллекции для организации рецептов (только свои)
- История просмотров и избранное
- Экран настроек (порции по умолчанию, TTL кэша, офлайн-режим, автосохранение истории)
- Полная работа без интернета: все пользовательские данные и кэш доступны офлайн
- Фоновая очистка устаревшего кэша и предзагрузка избранных рецептов (WorkManager)
- Реактивный UI на Flow/StateFlow/SharedFlow
- Тесты на ключевую функциональность (бизнес-логика, ViewModel, Room, Worker)


## Соответствие требованиям финального проекта

### 1. Экран настроек
Реализован `SettingsScreen` с настройками:
- Количество порций по умолчанию (используется при создании рецепта)
- Время жизни кэша (часы)
- Офлайн-режим (запрещает сетевые запросы)
- Автосохранение истории

### 2. Экран истории / recent
`HistoryScreen` – список ранее просмотренных рецептов (API). Поддерживается добавление в избранное, навигация к деталям. Управляется настройкой автосохранения.

### 3. Персональный раздел с пользовательскими данными
- **Мои рецепты** (`MyRecipesScreen`) – создание, редактирование, удаление собственных рецептов с ингредиентами и шагами.
- **Коллекции** (`CollectionsScreen`, `CollectionDetailScreen`) – группировка своих рецептов.

### 4. Новые законченные пользовательские сценарии

1. **Создание собственного рецепта** (существенный)  
   Пользователь заполняет форму (название, описание, ингредиенты, инструкции), рецепт сохраняется в Room. Можно добавить фото из галереи. Рецепт доступен в разделе «Мои рецепты» и в коллекциях.

2. **Организация рецептов в коллекции** (существенный)  
   Пользователь создаёт коллекцию (имя, описание, иконка-эмодзи) и добавляет в неё свои рецепты. Исключено дублирование. Коллекции видны в отдельном списке, из них можно перейти к деталям рецепта.

### 5. Локальное хранение
- **Room** – все пользовательские данные: `UserRecipe`, `Collection`, `Favourite`, `History`, кэш поиска.
- **DataStore** – только настройки (`AppSettings`).

### 6. Offline-first
- При включении офлайн-режима (или отсутствии сети) поиск и детали рецептов API недоступны, но приложение продолжает работать: доступны кэшированные результаты, свои рецепты, коллекции, избранное, история.
- Сетевые запросы в `RecipeBookViewModel` проверяют настройку `offlineModeEnabled` и при необходимости возвращают локальные данные.

### 7. Фоновая обработка (WorkManager)
- `CacheCleanupWorker` – периодически удаляет кэш старше TTL (из настроек).

### 8. Тесты
- **Unit-тесты:** `UserRecipeRepositoryTest` (сохранение, удаление, Flow), `CollectionsViewModelTest` (начальное состояние, создание, добавление).
- **Инструментальные тесты:** `CacheCleanupWorkerTest` (удаление старых записей), `UserRecipeDaoTest` (вставка и получение данных Room).
